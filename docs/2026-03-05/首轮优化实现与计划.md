# 首轮优化实现记录（2026-03-05）

## 1. 背景与目标

本轮目标聚焦以下问题：

1. 将后端默认大模型配置固定为 right.codes/codex，模型为 gpt-5.2，并兼容历史字段。
2. 将 Flow Canvas / Excalidraw 的“假流式”改为更真实的流式增量渲染。
3. 对前后端接口进行自测驱动优化，降低超时与异常，修复数据兼容问题。
4. 提升图形生成完整度与可读性（尤其 Excalidraw 连线缺失、节点类型异常、画布截断感）。

### 2.2 Flow 生成链路（真流式 + 容错 + 结构规范化）

涉及文件：
- `backend/app/api/chat_generator.py`
- `backend/app/services/chat_generator.py`
- `frontend/lib/store/useArchitectStore.ts`
- `frontend/components/ArchitectCanvas.tsx`

关键实现：

1. 流式失败切换（Failover）：
   - 按候选配置顺序尝试，首 token 超时触发切换。
   - 对上游错误进行分类（限流、鉴权、超时、无效载荷等），便于策略化处理。
2. 更真实的增量输出：
   - 缩短 partial 解析间隔，优先输出可解析的局部节点/边。
   - 增加 token batching，减少 SSE 事件风暴和前端渲染抖动。
3. Partial 数据清洗与归一化：
   - 未知 node type 自动映射到前端支持类型。
   - 对 start/end/decision/task 自动推断 shape。
   - 缺失坐标时给出稳定 fallback 坐标，保证画布可渲染。
4. 前端状态更新节流：
   - generation log 与聊天状态更新做节流，降低高频 setState 压力。
   - 非增量生成前先清空画布，避免历史残留污染。
5. 画布交互稳定性：
   - 生成中抑制自动 `fitView`，结束后再统一适配，避免流式阶段视图抖动。
   - 固化节点/边类型映射常量，兼容 legacy edge type 别名。

关键实现：

   - 首 token 超时保护。
   - token batching 降低 SSE 频率。
   - partial 解析更早触发，元素更早上板。
2. 连接完整性兜底（非掩盖式质量防线）：
   - 若 AI 输出无有效箭头/连线，基于图形读取顺序自动补最小连接集，避免“孤岛图”。
3. 视觉稳定性改进：
   - 前端 `ExcalidrawBoard` 增加 scene sanitize 与边界校验。
   - 生成完成后 `scrollToContent + fitToViewport`，减少“被截断”的观感。

4. 上传流式体验修复：
   - 逐元素上板，结束事件缺失时仍可安全收尾。
   - 错误信息展示更明确，避免静默失败。
5. Vision 侧稳态优化：
   - 长耗时分析增加心跳 progress 事件，降低前端长时间无响应感。
   - 视觉模型温度和 token 上限调优（更稳定 JSON 输出，降低超长尾时延风险）。

## 4. 下一步计划（第二轮持续优化）

### 4.1 回归与压测闭环（先验证再优化）

1. 重跑 targeted regression：
   - chat stream / excalidraw stream / vision stream-v2 / multipart validation
2. 重跑全量前后端 smoke，产出新版报告并与基线对比。
3. 引入关键时延指标看板：
   - TTFB、首元素时间、完成总时长、超时率、失败率。

### 4.2 真流式体验继续深化（核心）

1. Flow：
   - 进一步降低 partial merge 的等待阈值，确保用户输入后尽快看到首批节点。
   - 优化前端 incremental merge 策略，减少重排导致的“跳动”。
2. Excalidraw：
   - 提升增量解析器对不完整 JSON 的容忍度。
   - 继续优化“流式上板 vs 最终收敛”策略，减少尾段停顿。

### 4.3 Flow Canvas 架构图专项（补充）

1. 架构图生成从“流程图规则”中解耦，建立 architecture 专用策略：
   - technical / deployment / domain / business / layered 分类型提示词与后处理。
2. 引入 architecture 节点库规范（Node Catalog）：
   - 统一 category 到节点形态映射（service/api/gateway/database/cache/queue/storage/client/platform/observability/security/network）。
   - 保留 frame/layerFrame 用于分层与分组背景，组件节点使用语义化类型渲染。
3. 引入 Layering + Grouping 策略：
   - layer 负责纵向语义分层，group 负责层内集群（域、子系统、可用区、业务域）。
   - 自动生成 group 标识与视觉分组，避免大图“平铺无结构”。
4. 架构图边连接质量专项：
   - source/target 采用多别名解析（id/label/slug）避免“有边定义但无法挂接”。
   - 无边时按层间依赖自动补全最小可读连通骨架。
5. 架构图样式升级：
   - 增强节点图标体系与卡片视觉层次（状态色、语义色、技术栈标签、可读性优先）。
   - 改善超大图下的可读性与交互稳定性（缩放、视口适配、滚动冲突治理）。

### 4.4 复杂度与美观度优化（Prompt + 结构规则双轮驱动）

1. 建立复杂度分级策略（small/medium/large），按场景动态控制节点上限和细节深度。
2. 针对 React Flow 自定义节点库建立生成约束模板（字段规范、节点类型白名单、边类型规则）。
3. 对 Excalidraw 增加布局后处理规则：
   - 防重叠、连线最短路径倾向、关键节点层次保持。
4. 架构图取消固定上限思路：
   - 不设硬编码节点数量/嵌套深度/画板尺寸上限。
   - 改为分片生成 + 分层聚合 + 视口分段渲染，保障超大拓扑可持续构建。

### 4.5 协议重构：标准化 DSL + 分片增量输出（新增）

1. 建立标准化 DSL 原子节点协议（Architecture DSL v1）：
   - 原子对象：layer / group / node / edge / annotation。
   - 每个原子对象必须具备稳定 id、type、parent/group 关系、最小渲染字段。
2. 流式输出切换为分片结构化模式：
   - 支持 `layer` 分片、`node` 分片、`edge` 分片独立上板。
   - 前端在接收 JSON/XML 片段时即可执行局部 DOM 挂载与 Canvas 路径绘制，不等待全文完成。
3. 参考 Draw.io 的 mxGraphModel 分层单元组织原则：
   - 采用“稳定 cell/node id + 增量 append + 局部重算”思想，避免全量重排。
4. 保持与现有 SSE 协议兼容：
   - 过渡期同时支持 `[PARTIAL_NODE]/[PARTIAL_EDGE]` 与 DSL 事件，逐步平滑升级。

### 4.6 前端可视化验收（Playwright + Puppeteer）

1. 每轮优化后固定执行：输入 -> 生成 -> 截图 -> 结构检查。
2. 覆盖两类画布：
   - Flow Canvas：验证首节点出现时间与连通性。
   - Excalidraw：验证首元素出现时间、连接完整性、视口是否完整显示。
3. 强制增加像素级回归：
   - 快照对比、布局偏移监测、滚动条冲突检测、组件折叠冲突检测。
4. 形成图像化验收档案（`output/playwright/`）并关联接口日志溯源。

### 4.7 多智能体协同执行机制（新增）

1. 全栈架构师 Agent：
   - 负责后端协议、数据模型、前端节点渲染契约一致性与性能预算。
2. 视觉算法专家 Agent：
   - 负责布局可读性、分层分组、拥挤度与连线交叉治理、视觉质量规则。
3. 流式交互专家 Agent：
   - 负责 SSE 分片策略、首包/首节点时延优化、前端增量渲染调度与抖动治理。
4. 执行闭环：
   - 测试 -> 发现问题 -> 根因定位 -> 热修复 -> 回归 -> 再迭代，直至达标。
5. 约束：
   - 严禁掩盖错误；必须修根因并通过自动化验证证明。

### 4.8 目标验收标准（第二轮）
