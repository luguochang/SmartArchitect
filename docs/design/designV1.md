# SmartArchitect AI: 开发者职场生产力与汇报专家

## 1. 项目概览 (Project Overview)

### 1.1 背景

开发者在进行 PPT 述职汇报、技术博客撰写或工作总结时，经常面临“架构图难画”、“总结没亮点”、“图表不专业”的困扰。SmartArchitect 旨在通过 AI 帮助开发者将技术资产转化为高质量的视觉汇报材料。

### 1.2 核心愿景

* **复刻即效率**：秒级将流程图截图、手绘白板转化为可编辑、高颜值的 React Flow 节点。

* **重构即思考**：在复刻基础上，由 AI 指出架构瓶颈并提供基于专家经验的优化版本。

* **汇报即产出**：不仅输出图片，更输出“架构图 + 业务逻辑 + 汇报大纲 + 演讲稿”的完整包。

## 2. 技术架构 (Technical Architecture)

* **前端**: Next.js 14 + React Flow (负责高交互画布) + Zustand (状态管理)。

* **后端**: FastAPI + Python (OpenCV 处理 + RAG 引擎 + Gemini 2.5 Flash)。

* **数据层**: 支持 PDF/Markdown/Git Log 的多模态数据解析，为汇报提供事实支撑。

## 3. 核心功能设计 (Feature Specification)

### 3.1 AI 智能复刻 (The Core: Vision-to-Graph)

* **多模态提取**：识别图片中的形状、文字、连线方向及其嵌套层级。

* **结构化还原**：将识别出的内容 1:1 还原为 React Flow 节点。支持自动排版算法（Dagre/ELK），解决截图比例失调问题。

### 3.2 汇报级风格化 (Presentation Enhancement)

* **一键主题**：内置“大厂述职”、“科技极简”、“手绘风格”等 10+ 套汇报专用主题。

* **叙事标注**：AI 自动在图中添加气泡，解释关键技术决策（如：“引入消息队列解决流量突增”）。

### 3.3 Prompter 提示词系统 (Expert Prompts)

* **自定义 Prompt**：用户可以保存自己的绘图逻辑（如“所有数据库必须标记为红色”，“连线必须标注 QPS”）。

* **场景预设库**：

  * **架构重构模式**：输入图片，AI 自动生成“解耦后”的优化建议。

  * **安全加固模式**：自动识别单点故障并增加高可用组件。

  * **白板美化模式**：针对手机拍摄的模糊照片进行极致降噪与矢量化。

### 3.4 知识库集成 (RAG-Driven Presentation)

* **文档阅读**：支持读取设计文档 (PDF/Docx/MD) 作为汇报素材。

* **Context 感知生成**：AI 在生成 PPT 大纲时，会引用文档中的具体业务指标和背景，避免生成空洞的技术名词。

### 3.5 叙事脚本与 PPT 生成

* **演讲稿生成**：AI 根据架构变动，自动生成 30s/2min/5min 三个版本的汇报话术。

* **多端同步**：实时导出 PPT (pptx) 模板 or Slidev (Markdown PPT) 源码。

### 3.6 人工干预与实时编辑

* **全交互画布**：AI 识别后，用户可自由拖拽、修改连接线路由、双击编辑属性。

* **代码双向绑定**：修改画布，右侧生成的 Mermaid.js 或 Terraform 代码实时更新。

## 4. 技术规格 (Technical Specs)

### 4.1 视觉与逻辑解析

* **Gemini 视觉引擎**：捕捉图中微小的层级嵌套和逻辑流向。

* **RAG 索引**：使用轻量级向量数据库存储用户上传的项目文档，供 AI 在生成汇报大纲时调用。

## 5. 项目路线图 (Roadmap)

* [ ] **Phase 1: 核心复刻与画布 (MVP)** - 实现 React Flow 画布与“截图 -> 节点”的基础路径。

* [ ] **Phase 2: Prompter 专家库与风格系统** - 开发预设 Prompt 库及汇报主题切换功能。

* [ ] **Phase 3: 文档 RAG 与汇报生成** - 接入 PDF/Markdown 读取功能，实现架构图到 PPT 演讲稿的转化。

* [ ] **Phase 4: 演进视图与协作** - 支持架构版本对比（Diff View）及博客/文档外链动态预览。
